# 🚀 灵感厨房 (Inspiration Kitchen) 开发路线图 (2026 版)

本项目遵循 **“先简单、后丰富”** 的原则，从一个基于 Prompt 的轻量工具逐步演进为连接万物的智能厨房大脑。

---

## 📊 1. 阶段总览

| 阶段 | 版本 | 核心技术 | 核心交付 | 目标 |
| :--- | :--- | :--- | :--- | :--- |

## 🛠 2. 详细执行路径

### 🟢 第一阶段：点火 (V1.0) - Prompt 为王
**核心目标**：验证 AI 生成中式菜谱的逻辑。
* **前端 (Flutter)**: 
    * 简单高效的食材标签输入组件。
    * 结构化菜谱卡片展示（支持 Markdown 渲染）。
* **后端 (Python/FastAPI)**:
    * 调用 DeepSeek-V3 或 Gemini 1.5 Flash。
    * **重点**：编写 System Prompt，强制大模型开启 **JSON Mode** 返回结构化数据。
* **存储**: 使用本地缓存（Shared Preferences）记住口味偏好。

### 🟡 第二阶段：连接 (V1.2) - 引入 MCP
**核心目标**：让 AI 具备“感知”库存和“调用”外部服务的能力。
* **核心技术：MCP (Model Context Protocol)**:
    * **Inventory MCP Server**：让 AI 能实时读取你 App 里的食材库存。
    * **Delivery MCP Server**：连接生鲜电商 API，获取实时价格。
* **功能实现**:
    * **“差一味”补全**：AI 自动计算“现有食材 + 1件缺省食材”产生的最优解。
    * **一键买齐**：点击缺省食材，直接跳转美团/饿了么。



### 🟠 第三阶段：进化 (V1.5) - 多模态视觉
**核心目标**：降低录入成本，提升 App 的“黑科技”感。
* **Vision AI 接入**:
    * 使用 Gemini 1.5/2.0 Pro 的多模态能力。
    * 用户拍摄冰箱内部，AI 自动识别食材种类及其剩余量（如：半个洋葱）。
* **云端化 (Supabase)**:
    * 接入 Supabase Auth 实现跨端登录。
    * 使用云数据库同步冰箱库存。

### 🔴 第四阶段：深度 (V2.0) - RAG 与社区
**核心目标**：建立专业壁垒与用户粘性。
* **RAG (检索增强生成)**:
    * 挂载精品菜谱库与营养学向量数据库。
    * 消除 AI 瞎编调料比例的问题，提供精确到“克”的专业指导。
* **社区化**:
    * **灵感广场**：基于 Supabase 实现实时动态流，分享 AI 菜谱的实践成品。
    * **AI 进化循环**：根据用户反馈（如“太咸了”），自动调整全局 Prompt 策略。

## 💻 3. 技术栈建议

* **App 端**: Flutter 3.x (Dart)
* **逻辑层**: Python 3.10+ (FastAPI)
* **AI 引擎**: DeepSeek-V3 (推理) + Gemini 1.5 Flash (视觉)
* **连接标准**: **MCP SDK** (Anthropic 开源标准)
* **后端服务**: Supabase (数据库、鉴权、向量存储)
